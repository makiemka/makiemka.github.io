Tornado

Описание

Tornado — это расширяемый асинхронный веб-сервер и фреймворк, который решает проблему C10k. То есть при правильной настройке он может работать с 10 000+ одновременных соединений. Это делает его отличным инструментом для создания приложений, требующих суперпроизводительности при работе с операциями ввода/вывода и поддержки огромного количества одновременных соединений.

Изначально фреймворк был создан для использования в проекте FriendFeed, который в 2009 году приобрела компания Facebook, после чего исходные коды Tornado были открыты.

Особенности

Асинхронный режим работы

Использование веб-сокетов

Поддержка схем аутентификации и авторизации сторонних производителей.

Преимущества

Может масштабироваться до десятых тысяч открытых соединений, что достигается с помощью неблокирующего ввода-вывода

Позволяет работать с базами данных через сторонние библиотеки

Поддерживает разные веб-серверы и предлагает утилиты для работы с HTTP-метаданными

Обладает высокой производительностью и масштабируемостью, гибкостью и модульностью

Бесплатен и имеет открытый код

Поддерживается большим сообществом разработчиков.

Недостатки

Сложный для начинающих из-за асинхронности и потенциальных проблем совместимости из-за интеграции с asyncio.

Где используется

Используется в создании веб-приложений с большой нагрузкой, включая Facebook и API для Twitter.
